# 智能镜像优化系统 (Client-aware Transfer System)

本系统旨在通过感知客户端能力来优化容器镜像的传输效率，主要目标是根据不同客户端的硬件配置、网络状况等因素，为每个客户端定制最适合的压缩算法和传输策略。

## 核心特性

1. **客户端能力探测**：探测客户端的CPU性能、网络带宽、解压能力等
2. **自适应压缩决策**：根据客户端能力和镜像特征选择最佳压缩算法
3. **智能缓存机制**：缓存常用镜像的压缩版本，提高重复拉取效率
4. **差异化传输优化**：根据客户端历史行为优化传输策略

## 使用方法

### 1. 客户端拉取脚本（推荐）

客户端在拉取镜像时使用专用脚本，该脚本会自动检查和更新节点能力画像：

```bash
# 拉取镜像（自动处理能力画像）
python client_pull_script.py nginx:latest
```

脚本会自动：
- 检查本地是否存在有效的节点能力画像
- 如果不存在或已过期（默认24小时），则执行探针测试
- 使用最新的能力画像请求优化服务
- 执行镜像拉取

### 2. 手动注册节点能力

也可以手动执行一次性探针测试并注册节点能力：

```bash
# 执行探针测试并注册能力
python main.py --register

# 使用已注册的节点能力优化镜像
python main.py --node-id <NODE_ID> nginx:latest
```

### 3. 直接优化镜像（不使用能力画像）

可以直接运行优化服务，它会在本地执行探针测试：

```bash
# 直接优化镜像（每次都会执行本地探针）
python main.py nginx:latest
```

## 工作流程

1. **能力画像检查**：客户端拉取脚本检查本地是否有有效的节点能力画像
2. **探针测试**：如需要，执行一次性探针测试收集客户端能力
3. **能力注册**：将客户端能力注册到系统中，并保存到本地
4. **镜像分析**：分析目标镜像的特征，如文件类型分布、大小分布等
5. **压缩决策**：结合客户端能力和镜像特征，选择最佳压缩算法
6. **缓存检查**：检查是否已有相应压缩版本的缓存
7. **传输优化**：制定最优传输策略，如差分传输、分块传输等
8. **结果输出**：输出优化结果和建议的拉取方式

## 模块说明

### 客户端探针模块 (client_probe.py)
负责探测客户端的各种能力指标：
- CPU性能评分
- 网络带宽
- 解压缩速度
- 系统硬件信息

### 镜像探针模块 (image_probe.py)
分析Docker镜像的特征：
- 层(layer)的数量和大小
- 文件类型的分布情况
- 文件大小分布
- 可压缩性评估

### 决策引擎模块 (decision_engine.py)
根据客户端能力和镜像特征做出压缩决策：
- 预测不同压缩算法的效果
- 计算总体传输成本（压缩+传输+解压时间）
- 选择最优压缩算法

### 缓存池模块 (cache_pool.py)
管理压缩后的镜像缓存：
- 检查缓存命中情况
- 存储和检索压缩后的镜像数据
- 缓存淘汰策略

### 传输优化模块 (transmission_optimizer.py)
优化传输过程：
- 差分传输（只传输变更部分）
- 分块传输
- 带宽自适应调整

### 能力注册模块 (capability_registry.py)
管理客户端能力画像：
- 节点能力画像的注册和存储
- 节点ID生成
- 能力画像查询

## 技术细节

### 压缩算法支持
- gzip：通用压缩算法，兼容性好
- zstd：现代压缩算法，高压缩比和高速度
- lz4：极快的压缩算法，低CPU消耗

### 成本模型
系统通过以下公式计算每种压缩算法的总成本：
```
总成本 = 压缩时间 + 传输时间 + 解压时间
```

其中各项时间根据客户端能力和镜像特征进行预测。

### 节点能力画像
为避免频繁执行探针测试，系统引入了节点能力画像机制：
- 每个客户端有一个唯一的节点ID
- 节点能力画像包含CPU、网络、解压能力等信息
- 画像默认有效期为24小时
- 过期后自动重新测试更新
