# 容器友好的智能镜像优化系统

## 设计理念

本系统针对容器化环境的特殊约束进行了专门设计，确保在真实生产环境中能够有效运行。

## 核心设计原则

### 1. 无侵入性
- 不需要客户端运行额外的服务
- 不改变标准的Docker pull语义
- 兼容现有的容器编排系统

### 2. 被动探测为主
- 通过观察网络行为构建客户端画像
- 支持节点级画像继承
- 支持一次性探针注册机制

### 3. 自适应优化
- 根据观察到的行为动态调整传输策略
- 支持传输过程中的实时优化

## 四层探针架构

### 1. 镜像内容探针 (Image Probe)
- 分析镜像本身的可压缩性特征
- 为所有客户端提供统一的输入
- 不依赖客户端环境

### 2. 网络被动探针 (Network Passive Probe)
- 通过观察传输行为构建网络画像
- 支持节点级画像继承
- 适用于所有容器化环境

### 3. 预注册能力探针 (Pre-registered Capability Probe)
- 一次性执行探针并注册能力画像
- 提供精确的客户端能力数据
- 无需持续运行服务

### 4. 可选主动探针 (Optional Active Probe) [已废弃]
- 仅在特殊环境中考虑使用
- 需要客户端运行服务，不推荐

## 使用方法

### 容器化环境（推荐）
```bash
# 使用客户端IP进行被动探测
python main.py 192.168.1.100
```

### 预注册能力模式
```bash
# 首次注册节点能力（在客户端执行一次）
python client_agent.py

# 服务端使用预注册能力
python main.py node:<NODE_ID>
```

### 本地测试环境
```bash
# 使用本地探测
python main.py
```

## 节点画像管理

### 节点标识
- 使用IP地址和User-Agent生成唯一节点ID
- 支持画像在同节点容器间继承
- 自动清理过期画像

### 能力注册
- 一次性执行探针测试
- 注册能力画像到中央注册中心
- 后续请求基于预注册画像优化

### 画像更新
- 基于传输行为观察持续优化
- 使用移动平均算法平滑指标变化
- 支持画像置信度评估

## 系统优势

### 兼容性
- 完全兼容标准Docker工作流
- 无需修改客户端配置
- 支持主流容器编排平台

### 准确性
- 基于真实传输数据构建画像
- 支持多维度能力评估
- 持续学习优化

### 性能
- 首次注册后获得持续优化
- 避免频繁探测开销
- 支持大规模并发

### 安全性
- 无需客户端运行持续服务
- 降低安全风险
- 减少资源消耗

## 架构图

```
标准容器环境:
Docker Client → Registry → 返回优化镜像
     ↑               ↓
   被动观察网络行为  节点画像库

预注册能力环境:
客户端执行探针 → 注册中心 → Registry使用预注册画像 → 返回优化镜像
```
