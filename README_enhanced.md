# 智能镜像优化系统增强版

## 新增功能

### 一次性探针执行 + 能力画像注册机制

系统现在支持更符合实际生产环境的节点能力管理机制，无需持续运行客户端服务。

## 使用方法

### 1. 注册节点能力（一次性执行）

在客户端环境中执行一次探针并注册能力：

```bash
python client_agent.py
```

这将：
- 执行本地探针测试（CPU、网络、解压性能等）
- 生成唯一的节点ID
- 注册节点能力画像到系统中

### 2. 查看已注册节点

```bash
python client_agent.py list
```

### 3. 使用预注册能力进行镜像优化

在服务端使用已注册的节点ID进行优化：

```bash
python main.py node:<NODE_ID>
```

其中 `<NODE_ID>` 是注册时生成的节点ID。

### 4. 被动探测模式（容器环境默认）

对于容器化环境，系统仍然支持被动网络探测：

```bash
python main.py <CLIENT_IP>
```

## 系统架构

### 三层探针架构

1. **镜像内容探针**（Image Probe）
   - 分析镜像静态特征
   - 为所有客户端提供统一输入

2. **网络被动探针**（Network Passive Probe）
   - 通过观察网络行为构建画像
   - 适用于容器化环境

3. **预注册能力探针**（Pre-registered Capability Probe）
   - 一次性执行探针并注册
   - 提供精确的能力画像

### 工作流程

```
首次使用：
客户端 → 执行探针 → 注册能力画像

日常使用：
客户端 → 请求镜像 + 节点ID
服务端 → 查找能力画像 → 提供优化镜像
```

## 优势

### 1. 无持续服务依赖
- 不需要客户端持续运行服务
- 降低资源消耗和安全风险

### 2. 灵活的部署方式
- 支持多种环境（容器、物理机、虚拟机）
- 兼容现有Docker工作流

### 3. 持续优化
- 能力画像可更新
- 支持定期重新注册以保持准确性
